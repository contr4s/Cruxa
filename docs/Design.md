# UI/UX & Design System

## 🎨 Дизайн Концепция
На проекте используется единая дизайн-система **Material Design 3 (M3)** для обеспечения максимальной скорости разработки.

- **Web:** Использование библиотеки [MUI (Material-UI) 9](https://mui.com/) с кастомной тёмной темой.
- **Mobile:** Родной фреймворк Flutter предоставляет полноценную поддержку [Material 3](https://m3.material.io/). *(не начато)*

## 🔤 Типографика
- **Шрифт:** `'Segoe UI', system-ui, -apple-system, sans-serif`
- **Размеры:**
  - Заголовок страницы (h1): 1.6rem, 800 weight
  - Заголовок секции (h2): 1.4rem, 700 weight
  - Подзаголовок (h3): 1.15rem, 600 weight

## 🧩 Ключевые UI-компоненты (из мокапа)

### 1. Навигация (Sidebar)
- Левое фиксированное меню на всех страницах (кроме Landing)
- Иконка + текст для каждого пункта
- Активный пункт подсвечивается `--primary`
- Аватар пользователя с выпадающим меню
- Mobile: нижняя навигационная панель (tab bar)

### 2. Хидер страницы
- Логотип "▲ Крукса" слева
- Навигационные ссылки по центру
- Аватар и уведомления справа
- Кнопка "Назад" на детальных страницах

### 3. FAB (Floating Action Button)
- Круглая кнопка "＋" в правом нижнем углу
- Используется на страницах Climber и Gym Detail
- Открывает форму создания тренировки / добавления пролаза

### 4. Карусель (Carousel)
- Стрелки "‹" / "›" по бокам
- Слайд-индикаторы (точки)
- Адаптивная высота (aspect-ratio: 16/10 для gym, 3/4 для route, 4/3 для post)
- Переиспользуется на всех страницах с медиа

### 5. Таблицы с сортировкой
- Стрелки "↕" / "▲" / "▼" в заголовках столбцов
- Сортировка по клику на заголовок
- Чередование фона строк (surface / surface2)
- Выравнивание числовых колонок по правому краю
- Адаптивный horizontal scroll (min-width: 700px)

### 6. Фильтры
- Комбобоксы (select) для выбора: тип трассы, статус, рутсеттер, сектор, теги, город
- Range-слайдеры для грейда (min/max)
- Опция "Ещё фильтры" (collapsible)

### 7. Пагинация
- Номера страниц кнопками
- Кнопки "‹" / "›" для prev/next
- Текущая страница с highlight `--primary`
- Информация "1–5 из 20"

### 8. Карточки статистики (Stat Cards)
- Иконка + число до 4 знаков + подпись
- Используются на всех дашбордах (профиль, зал, админка, рутсеттер)

### 9. Графики и чарты
Реализованы через Chart.js + react-chartjs-2:
- **Комбинированный график** (Крускор + грейд): двухосный, с областями, градиентом, точками данных
- **Радар навыков** (6 категорий: баланс, динамика, кампус, силовой, статика, техничный)
- **Пирамида грейдов** (горизонтальные бары, от низшего к высшему, с кол-вом трасс)
- **Пончик-диаграмма** (типы пролазов: Flash, Redpoint, Onsight, Attempt, Project, Repeat)
- **Консенсус-график** (распределение голосов за грейд на странице трассы)
- **График недельной активности** (мини-бары на GymAdmin/Admin)

### 10. Календарь-хитмап (Heatmap)
- Сетка 7 колонок (дни недели)
- Заливка ячеек с переменной яркостью (`filter: brightness(0.7 + intensity × 0.5)`)
- Навигация по месяцам ("‹" / "›")
- Числа дней для заполненных, пустые для пропущенных

### 11. Бейджи и чипсы
- `badge-primary`: зелёный — активное, on
- `badge-secondary`: жёлтый — предупреждение, pending
- `badge-gray`: серый — архив, off
- `badge-green`: тёмно-зелёный — подтверждено
- Размер: 0.75rem, uppercase, border-radius: 12px

### 12. Аватар
- Круглый, 32-40px
- Первая буква имени + случайный цвет фона
- Опционально: статус-точка (online/offline)

### 13. Empty State / Нет данных
- Иконка + заголовок + описание + CTA-кнопка
- Пример: 🪢 "Нет подходящих трасс" + "Попробуйте изменить параметры фильтра"

### 14. Toast / Snackbar
- Реализован через notistack (`NotificationProvider`)
- Позиция: bottom-center
- Типы: success (primary), error (red), info

### 15. Skeleton / Loading
- Реализованы кастомные Skeleton-компоненты
- Shimmer-анимация на карточках, таблицах, графиках

### 16. Confirm-диалог
- Реализован через MUI Dialog (ConfirmDialog)
- Заголовок + описание + Cancel/Confirm кнопки
- Danger-действия с красной кнопкой подтверждения

## 📱 Ключевые Экраны — Mobile (Flutter)
1. **Лента (Feed):** Агрегированные карточки тренировок друзей (Posts). Карусель выбранных медиа, текстовое описание, сводка пролазов, кнопки лайков и комментариев.
2. **Скалодромы (Gyms):** Список и Карта (Google/Yandex Maps). Страница зала с фильтрами трасс по тегам и категориям. Большая кнопка (FAB) "Начать тренировку" (создает Draft Post).
3. **Карточка трассы (Route Details):** Детальная информация о трассе. Большое фото, цвет зацепок, теги, среднее значение рейтинга и лента публичных отзывов.
4. **Активная тренировка и QR-сканер:** Режим черновика. Присутствует FAB запуска камеры для моментального сканирования QR-кода трассы на стене зала.
5. **Логирование пролаза (Add Ascent):** Модальный Bottom Sheet при выборе трассы. Выбор стиля прохождения (Flash, Redpoint и т.д.). Прикрепление медиа к пролазу.
6. **Карточка отзыва о трассе (Route Review):** После прохождения трассы (или отдельно) пользователь может оставить рейтинг (1-5), приватные заметки и публичный отзыв.
7. **Завершение тренировки (Publish Workout):** Экран курации контента. Галочками выбираются лучшие фото/видео со всех пролазов, пишется общее описание тренировки, настраивается приватность.
8. **Профиль (Profile):** Крускор, аналитика, цели, достижения, календарь активности, история тренировок, список подписок/подписчиков.

## 💻 Ключевые Экраны — Web (React + Мокап)
Веб-версия (9 страниц мокапа, `docs/mockup/`) покрывает **4 роли и все ключевые сценарии**:

### 🏠 Landing Page (`landing.html`)
- **Роль:** Анонимный пользователь
- **Компоненты:** навигация (лого + "Войти" + "Присоединиться"), hero-секция с мокапом телефона, 6 карточек возможностей, статистика (10+ залов, 500+ скалолазов, 1000+ трасс, 24/7), CTA "Присоединиться бесплатно", footer
- **Анимации:** scroll-reveal, SVG-паттерны в hero

### 🧗 Climber Dashboard (`climber.html`)
- **Роль:** Скалолаз (Climber)
- **Компоненты:**
  - **Профиль:** аватар, имя, роль, город, статистика (трассы, макс грейд, подписчики, подписки), бейджи/достижения, кнопка "Редактировать"
  - **Стат-карточки:** Крускор, трассы, тренировки, залы
  - **Лучшие трассы:** топ-7 с грейдом, стилем, залом и рейтингом
  - **Календарь-хитмап:** навигация по месяцам, интенсивность
  - **Цели:** целевой грейд, трассы, залы с прогресс-барами
  - **Графики:** комбинированный Крускор+грейд, радар навыков (с переключением категорий), пирамида грейдов, пончик-диаграмма типов пролазов
  - **FAB "＋"** — создание тренировки

### 🗺 Gym Detail (`gym-detail.html`)
- **Роль:** Скалолаз (Climber)
- **Компоненты:** карусель фото зала, мини-карточки под каруселью (площадь, высота, год), шапка (название + рейтинг + ♪ избранное), адрес + метро, описание, часы работы, цены (разовое / абонементы 8 и 12), контакты (телефон, email, VK, Instagram, YouTube, сайт), статистика (активных трасс, ср. рейтинг, диапазон грейдов), база трасс с фильтрацией по типу/грейду/тегу/рутсеттеру и пагинацией, FAB

### 🧗 Route Detail (`route-detail.html`)
- **Роль:** Скалолаз (Climber)
- **Компоненты:** карусель медиа трассы, шапка (название + грейд), мета (зал, сектор, тип, дата), рутсеттер, консенсус-график голосования за грейд, средний рейтинг, теги (стиль), приватные заметки (textarea collapsible), кнопка "🧗 Пролез!", оценка сложности сообществом (распределение голосов), отзывы с аватарами + пагинация

### 📰 Post Detail (`post-detail.html`)
- **Роль:** Скалолаз (Climber)
- **Компоненты:** two-column layout (Instagram-стиль), переключение табов "Фото / Статистика", карусель медиа, карусель статистики (пончик, пирамида, радар), сводка (Крускор + ср. грейд + длительность + кол-во трасс), профиль автора + дата + видимость, описание тренировки, список пролазов, лайки (анимация heartBeat), комментарии с пагинацией и формой ввода

### 🛠 Routesetter Dashboard (`routesetter.html`)
- **Роль:** Рутсеттер
- **Компоненты:** стат-карточки (активных трасс, ср. рейтинг, всего пролазов), быстрый обзор (самый популярный, самый рейтинговый, новых за неделю), таблица "Мои трассы" с фильтрацией (статус, зал) и сортировкой (грейд, тип, дата, рейтинг, пролазы), кнопка "+ Новая трасса", отзывы на трассы (аватар + имя + текст + рейтинг), сводка отзывов (среднее, лучший, требует внимания), мои скалодромы (карточки залов с показателями), кнопка "+ Привязать зал"

### 🏢 GymAdmin Dashboard (`gymadmin.html`)
- **Роль:** Администратор скалодрома
- **Компоненты:** стат-карточки (трасс всего, активных, пролазов за месяц), активность за неделю (трассы, пролазы, отзывы, посетители), таблица "Все трассы зала" с фильтрацией (сектор, рутсеттер, статус) и кнопкой "+ Добавить", кнопка "📥 Экспорт", управление рутсеттерами (таблица: имя, активных трасс, ср. рейтинг, отвязать), редактирование профиля зала (название, описание, адрес, цены, часы, фото), кнопка "💾 Сохранить"

### ⚙️ Admin Dashboard (`admin.html`)
- **Роль:** Super Admin
- **Компоненты:** стат-карточки (скалодромов, трасс, рутсеттеров, пролазов), последние обновления (что где изменилось), топ залов по активности, таблица "Все скалодромы" с фильтрацией (город, статус) и пагинацией, колонки (зал, город, трасс, рутсеттеров, рейтинг, пролазов/мес, статус, действия), кнопка "📊 Экспорт данных", кнопка "+ Добавить зал"

## 📱 Адаптивность (Responsive)
- **Desktop (>1024px):** Full two-column/three-column layouts, сайдбар виден, таблицы полностью
- **Tablet (768-1024px):** Частичная адаптация, сайдбар сворачивается, сетки 2→1 колонка
- **Mobile (<768px):**
  - Вертикальная компоновка (flex-direction: column)
  - Нижняя навигация (tab bar) вместо сайдбара
  - Скрытые лейблы в навигации, только иконки
  - Таблицы с горизонтальным скроллом (min-width: 600px + overflow-x: auto)
  - Сетки 2→1 колонка

## 🏔 Цветовое кодирование трасс (Routes)
Ключевая фича на скалодромах — это цвет зацепок. В системе стандартизирована палитра из 11 цветов + мультиколор (см. таблицу выше).
- В карточке трассы `Route` обязательно использовать маркер соответствующего цвета в UI, чтобы скалолаз в зале мог легко визуально сопоставить телефон и стену.
- Цвет отображается в виде цветного квадрата/круга перед названием трассы.
- Фильтр по цвету зацепок реализован как ряд кликаемых цветных квадратов.
