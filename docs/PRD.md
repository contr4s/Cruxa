# Product Requirements Document (PRD)

## 📌 Цель Проекта
Создать социальную сеть для скалолазов (аналог Strava / Kaya Gyms), где пользователи могут находить скалодромы, отмечать пролазы (ascent logging) и соревноваться/общаться с другими участниками.

## 🎯 Целевая Аудитория
1. **Скалолазы (любители и профи)** – логирование тренировок, поиск скалодромов, социальное взаимодействие.
2. **Рутсеттеры (создатели трасс)** – отслеживание популярности своих трасс, получение фидбека (оценки).
3. **Владельцы скалодромов** – аналитика посещаемости, привлечение новых клиентов, актуальное расписание и база трасс.

## ✨ Ключевой Функционал MVP (Minimum Viable Product)
- **Каталог скалодромов:** 
  - Формируется автоматически (парсинг открытых API, таких как Яндекс Карты/2GIS).
  - Карточка скалодрома с адресом, контактами, списком трасс.
- **Журнал скалолаза (Ascent Log):** 
  - Возможность для пользователя отметить пройденную трассу.
  - Указание стиля прохождения (Flash, Onsight, Redpoint, Top Rope).
- **Социальная лента:** 
  - Лента событий друзей (кто, что, где пролез).
  - Лайки, комментарии к пролазам.
- **Профили пользователей:** 
  - Личная статистика (количество трасс, максимальный грейд).
  - Система подписок (Follow / Following).

## 🤖 Модуль парсинга (Требования)
- **Цель:** Исключить "проблему холодного старта". На этапе MVP база скалодромов должна быть собрана автоматически по России.
- **Инструменты:** Скрипты на Python/C# для сбора данных по ключевым словам "скалодром" (через Yandex Maps API/OSM/2GIS).
- **Регулярность:** Единоразовая первоначальная загрузка, далее — возможность ручного редактирования через Web-админку.
