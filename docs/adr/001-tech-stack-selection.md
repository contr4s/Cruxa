# ADR 001: Выбор основного технического стека

## Статус
Принято

## Контекст
Проект разрабатывается одним инженером и требует запуска на нескольких платформах (iOS, Android, Web) с надежным бекендом. Необходимо выбрать стек, который обеспечит высокую скорость разработки, надежность типов и масштабируемость, минимизируя время на поддержку.

## Рассмотренные варианты
- **Mobile:** React Native, Kotlin (Android) / Swift (iOS), Flutter.
- **Web:** Vue.js, Angular, React + TypeScript.
- **Backend:** Node.js (NestJS), Python (Django/FastAPI), C# .NET.
- **База данных:** MongoDB (NoSQL), MySQL, PostgreSQL (SQL).

## Решение
Был выбран следующий стек:
- **Backend:** C# ASP.NET Core Web API + Entity Framework Core.
- **База данных:** PostgreSQL (с перспективой PostGIS для гео-данных).
- **Web-платформа:** React + TypeScript + Material-UI (MUI).
- **Mobile App:** Flutter + Material 3.

## Обоснование
1. **Строгая типизация везде:** C#, TypeScript и Dart гарантируют защиту от многих runtime-ошибок, что критично для соло-разработчика.
2. **Flutter:** Позволяет написать единый код сразу для iOS и Android с нативной производительностью.
3. **C# .NET & EF Core:** Мощнейший ORM и enterprise-level фреймворк, обеспечивающий быструю разработку надежного API.
4. **PostgreSQL:** Структура данных (пользователи, скалодромы, посты, подписки) строго реляционная. Плюс поддержка JSONB для гибких форматов (расписания, цены).

## Последствия
- Потребуется поддержка трех разных языков (C#, TS, Dart), однако их схожий синтаксис (C-подобные) и строгая типизация смягчат контекст-свитчинг.
