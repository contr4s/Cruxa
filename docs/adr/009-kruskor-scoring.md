# ADR 009: Kruscore Scoring Algorithm

## Статус
Active

## Дата
2026-07-08

## Трек
B

## Контекст

Необходим композитный скор скалолаза (Kruscore), который:
- Отражает прогресс навыков с течением времени
- Учитывает стиль прохождения, тип трассы, грейд и объём
- Работает при холодном старте (без истории)
- Затухает при простое (decay)
- Поддерживает per-tag навыки для радара

## Решение

Elo-подобная вероятностная модель с доменными адаптациями.

### Базовая формула

```
E = 1 / (1 + 10^(-(θ − gradeIndex) / scale))
θ += K × avg(S − E)
```

### S (степень успеха)

**Lead:**

| Style | S |
|-------|---|
| Onsight | 1.0 |
| Flash | 0.9 |
| Redpoint | 0.75 |
| TopRope | 0.60 |
| Repeat | max(0.5, E) |
| Attempt | 0.0 |

**Bouldering:**

| Style | S |
|-------|---|
| Onsight | 1.0 |
| Flash | 1.0 |
| Redpoint | 0.70 |
| TopRope | — |
| Repeat | max(0.5, E) |
| Attempt | 0.0 |

### Scale (крутизна сигмоиды)

| Route.Type | base_scale | Attempt (×0.5) |
|------------|-----------|----------------|
| Bouldering | 100 | 50 |
| Lead | 75 | 37.5 |

### Radar Scale (для навыков)

Все типы: base_scale = **25**, Attempt = **12.5**

### Grade Reliability Weight

```
w = 1 − 0.3 × (|avgUserRating − GradeIndex| / 200) × min(1, ratingsCount / 10)
```

Пока нет рейтингов → w = 1.0.

### K-Factor

```
K = max(5, 32 / √C)
```

### Warm-start (калибровка)

Kruscore показывается только после **суммарного веса ≥ 15**.  
Вес одного асцента: Lead = **3**, Bouldering = **1**.  
Attempt и Repeat исключаются из калибровки.

При первом расчёте Performance Rating применяется ко **всем успешным асцентам разом**:

```
θ_i = gradeIndex_i + 50 × (S_i − 0.7)
θ₀ = weighted_avg(θ_i, weight_i)
C₀ = n (все асценты)
```

| Сценарий | S | weight | θ_i |
|----------|---|--------|-----|
| Lead Onsight 6C | 1.0 | 3 | 600 + 50×(1.0−0.7) = 615 |
| Bouldering Flash 5A | 0.9 | 1 | 460 + 50×(0.9−0.7) = 470 |
| Bouldering Redpoint 6C | 0.75 | 1 | 600 + 50×(0.75−0.7) = 602.5 |
| Lead TopRope 6C | 0.60 | 3 | 600 + 50×(0.6−0.7) = 595 |

### Online (последующие тренировки)

После калибровки каждая новая тренировка обрабатывается инкрементально:

1. **Decay** (перед батчем):
   - `decay = 2^(-daysSinceLast / 90)`
   - `θ −= 50 × (1 − decay) × 0.5`
   - `C ×= decay`

2. **Batch**:
   - `avgSurprise = sum(w × (S − E)) / sum(w)`
   - `θ += K × avgSurprise`
   - `C += sum(w)`

### Decay

Экспоненциальный, период полураспада = 90 дней.

| Простой | decay |
|---------|-------|
| 0 | 1.00 |
| 30d | 0.79 |
| 90d | 0.50 |
| 180d | 0.25 |
| 365d | 0.06 |

### Radar Skills

Per-tag изолированный Kruscore с scale=25:
- `θ₀ = baseline = (θ_global − 400) / 4`
- `C₀ = 1`
- Только ascents с тегом
- Те же S, decay, w_grade

## Последствия

- Warm-start устраняет произвольный калибровочный период
- Экспоненциальный decay предотвращает застой скора при простое
- Раздельные таблицы Bouldering/Lead отражают доменные различия
- Grade Reliability Weight готов к появлению пользовательских рейтингов
- Radar scale (25) делает различия навыков заметными на шкале 0–100
