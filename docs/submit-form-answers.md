Что за продукт и какую задачу решает:
Cruxa — социальная сеть и трекер активности для скалолазов (аналог Strava). Решает проблему разрозненности: в России нет единого инструмента для поиска скалодромов, логирования пролазов и отслеживания прогресса. Всё в одном месте: каталог 50+ городов, дневник тренировок, аналитика с собственной Elo-подобной системой рейтинга (Крускор) и социальная лента.

Использованные языки/фреймворки:
Backend: C# .NET 10, ASP.NET Core, Entity Framework Core, MediatR, FluentValidation, PostgreSQL.
Frontend: TypeScript 6, React 19, MUI 9, TanStack Query 5, Zustand 5, Vite 8, Chart.js.
Infra: Docker, Docker Compose, nginx.
Тесты: xUnit, Testcontainers, FluentAssertions.

Как применяли Kodik:
Использовал Kodik для разработки фронтенда: генерация React-компонентов, страниц, сервисного слоя, настройка TanStack Query и Zustand. Потратил все выданные токены на ускорение разработки UI.

Ключевой пользовательский сценарий:
Пользователь находит ближайший скалодром в каталоге (фильтр по городу, сортировка по расстоянию/трассам)
Приходит в зал, нажимает «Начать тренировку» — создаётся черновик
Логирует каждый пролаз: выбирает трассу, стиль прохождения (Flash/Redpoint/Onsight), указывает сложность
Завершает тренировку — пост публикуется в ленту
В профиле доступна аналитика: пирамида грейдов, радар навыков, календарь-хитмап, динамика рейтинга Крускор
